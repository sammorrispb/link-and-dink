# Running Focus

*The difference between coaching a player and coaching a session. One has a memory; the other doesn't.*

**Definition.** The longer-arc developmental thread a coach is helping a specific player work on *across* sessions — distinct from the in-the-moment feedback of any single session. A player's running focus might be "commit to a drop-or-drive decision before the ball comes," tracked and revisited over weeks.

**Why it matters.** Without a running focus, every session starts from zero — the coach reacts to whatever happens that day and the player never builds a through-line. With one, the coach is genuinely *developing* the player: each session is a chapter, not a standalone. For a recurring 1:1 or a returning small group, the running focus is what makes the coaching add up.

**Coaching it (you, the coach).** Write it down. After a session, note each player's running focus and any movement on it ([post-event debrief](../flows/post-event-debrief-flow)). Use it as an input when you [design the next session](../frameworks/session-design-framework) and when you [review a lesson plan](../flows/lesson-plan-review-flow). A coach whose running-focus notes "tell a story about each player" is a coach who's actually developing people.

> Every player has a thread you're pulling across sessions — track it, or you're starting over every time.
